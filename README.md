# Personal Finance Management App — Loglizer Anomaly Detection

This project demonstrates automated log anomaly detection using [Loglizer](https://github.com/logpai/loglizer) on logs generated by a personal finance management application.

---

## Architecture

```
Finance App Logs
      │
      ▼
logs/application.log
      │
      ▼
ml/parser.py  →  Log parsing & session grouping
      │
      ▼
ml/train.py   →  PCA model trained on normal logs
      │
      ▼
ml/detect.py  →  Anomaly detection on mixed logs
      │
      ▼
ml/report.py  →  reports/report.md + reports/anomalies.json
      │
      ▼
GitHub Actions Artifact
```

---

## Repository Structure

```
├── logs/
│   ├── normal.log          # Normal user behavior logs
│   ├── abnormal.log        # Simulated anomaly logs
│   └── application.log     # Mixed logs for detection
├── ml/
│   ├── generate_logs.py    # Generates all log datasets
│   ├── parser.py           # Log parser & session grouping
│   ├── train.py            # Trains PCA model on normal logs
│   ├── detect.py           # Detects anomalies, outputs JSON
│   ├── report.py           # Generates Markdown & JSON reports
│   └── requirements.txt    # Python dependencies
├── reports/
│   ├── report.md           # Markdown anomaly report
│   ├── anomalies.json      # Raw detection results
│   └── summary.json        # Summary for dashboards
└── .github/workflows/
    └── loglizer.yml        # CI/CD pipeline
```

## Anomaly Types Detected

| Anomaly | Description |
|---------|-------------|
| Brute-force login | 5+ consecutive failed login attempts |
| Server exceptions | Repeated NullPointerException, IOException, etc. |
| Database timeout | Multiple DB connection timeout errors |
| Unauthorized access | Repeated access to `/admin`, `/api/users` |
| Traffic spike | 100–500 rapid requests (DDoS-like pattern) |
| Slow responses | Requests taking 5000ms+ |

## Generate Logs

```bash
python3 ml/generate_logs.py
```

Produces:
- `logs/normal.log` — 100 normal user sessions
- `logs/abnormal.log` — 20 anomaly scenarios
- `logs/application.log` — mixed dataset for testing

---

## Run Anomaly Detection Locally

```bash
# 1. Train the model on normal logs
python3 ml/train.py

# 2. Detect anomalies in application logs
python3 ml/detect.py

# 3. Generate reports
python3 ml/report.py
```

Reports are saved to `reports/`.

---

## CI/CD Pipeline (GitHub Actions)

The workflow at `.github/workflows/loglizer.yml` runs automatically on every push to `main`/`master`.

Steps:
1. Checkout repository
2. Set up Python 3.10
3. Install dependencies (`ml/requirements.txt`)
4. Generate sample logs
5. Train the Loglizer PCA model
6. Run anomaly detection
7. Generate reports
8. Upload `reports/` as a GitHub Actions artifact (retained 30 days)

To trigger manually: go to **Actions → Loglizer Anomaly Detection → Run workflow**.

## Model

Uses **PCA (Principal Component Analysis)** from Loglizer. Trained exclusively on normal logs, it learns the expected event distribution per session. Sessions that deviate significantly from this distribution are flagged as anomalies.
